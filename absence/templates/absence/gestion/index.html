Les alertes : comment ça marche ?

Où les voir ? Menu Alertes (/scolarite/alertes/) pour la scolarité / admin. L’étudiant concerné les voit aussi sur l’accueil et Mes absences.

Que contiennent-elles ? Un message du type « un étudiant a trop d’absences non justifiées pour un cours », avec le nombre et le seuil.

Quand apparaissent-elles ?

  • Il existe un seuil (en démo : 3 absences non justifiées pour BD501), défini en base par année ou par cours.
  • Un enseignant enregistre des absences marquées non justifiées.
  • Dès que le nombre dépasse le seuil, le système enregistre une alerte dans l’historique.

Avec Oracle (scripts du dossier oracle/) : c’est la base qui crée l’alerte automatiquement (logique métier côté Oracle).

Avec SQLite sur votre PC : cette automatisation n’est pas reproduite ; pour la démo, vous pouvez ajouter des lignes manuellement dans l’admin Django → Historique alerte, ou tester sur Oracle.

Test guidé (recommandé) : dans un terminal, python manage.py seed_scenario_alerte puis suivez le texte affiché ou le fichier GUIDE_TEST_ALERTES.txt à la racine du projet (cours ALERT01, seuil 3, SQLite ou Oracle).
